# tools/yaml_to_csv.py

Converts Anki entry YAML files into Anki-importable CSV files.

YAML is the human-editable source of truth. CSV files are generated artifacts
and should be regenerated whenever a YAML file changes.

---

## Requirements

Python 3.10+ and PyYAML:

```bash
pip install pyyaml
```

---

## Usage

**Convert all entry YAML files in the repository:**

```bash
python tools/yaml_to_csv.py
```

**Convert a single file:**

```bash
python tools/yaml_to_csv.py --file note-types/Japanese_Mined_Vocab/entries/001_明日は遠足.yaml
```

The output CSV is written next to the YAML file with the same base name
(e.g. `001_明日は遠足.yaml` → `001_明日は遠足.csv`). Any existing CSV is overwritten.

---

## Repository structure

```
note-types/
  {NoteTypeName}/
    entries/
      entries-config.yaml        ← shared config for this Note Type (required)
      {deck-name}.yaml           ← entry source file (human-edited)
      {deck-name}.csv            ← generated by this tool (do not hand-edit)
```

---

## entries-config.yaml

Every `entries/` directory **must** contain an `entries-config.yaml` that
defines the canonical field list for its Note Type. The tool reads this file
to determine column order, which fields are required, and where the `Tags`
column is located.

**Schema:**

```yaml
notetype: YourNoteTypeName

# Fields in the order they should appear as CSV columns.
# The first field is always the ID field used for idempotent Anki imports;
# it must never be empty in any entry.
fields:
  - name: FieldName
    required: true   # tool aborts if this field is missing or empty in any entry
  - name: OptionalField
    required: false  # tool outputs an empty column if absent
```

**Example** (`note-types/Japanese_Mined_Vocab/entries/entries-config.yaml`):

```yaml
notetype: Japanese_Mined_Vocab

fields:
  - name: VocabID
    required: true
  - name: TargetWord
    required: true
  - name: Reading
    required: true
  - name: Meaning
    required: true
  - name: Sentence
    required: true
  - name: SentenceTranslation
    required: true
  - name: Notes
    required: false
  - name: Audio
    required: false
  - name: Tags
    required: false
```

The `Tags` field is special: its 1-based column position is automatically
used to emit the `#tags column:<N>` Anki directive. If your Note Type has
no `Tags` field, the directive is omitted.

## Entry YAML file format

```yaml
notetype: Japanese_Mined_Vocab          # must match entries-config.yaml
deck: Japanese::Todoku::明日は遠足       # target deck for this file

entries:
  - VocabID: 001_0001
    TargetWord: 始まる
    Reading: はじまる
    Meaning: to begin; to start
    Sentence: もう一時間めは<b>始まっています</b>。
    SentenceTranslation: The first period has already started.
    Notes: Intransitive verb. Transitive pair is 始める (to start something).
    Audio:
    Tags:
```

**Notes on field values:**

- HTML tags (e.g. `<b>`, `<br>`) are preserved verbatim.
- Trailing whitespace and newlines are stripped automatically, so YAML block
  scalars (`|`) work correctly without producing stray characters in Anki.
- Empty optional fields can be written as `FieldName:` (null) or omitted
  entirely; both produce an empty CSV column.
- **The ID field (first field) must always be quoted** to prevent YAML from
  misinterpreting it as a number. IDs like `001_0001` look like octal integers
  to YAML parsers and will be silently corrupted if unquoted. Always write:
  ```yaml
  VocabID: "001_0001"   # correct — quoted string
  VocabID: 001_0001     # wrong — YAML parses this as the integer 4097
  ```
  The tool aborts with a clear error if a non-string ID is detected.

## Validation rules

The tool aborts with a descriptive error message if any of these rules are
violated. No partial CSV is written.

| Rule | Behaviour |
|---|---|
| `entries-config.yaml` is missing | Abort |
| Entry file `notetype` does not match config | Abort |
| Entry file is missing the `deck` key | Abort |
| The ID field (first field) is empty in any entry | Abort |
| A `required: true` field is missing or empty in any entry | Abort |
| An entry contains a field not declared in `entries-config.yaml` | Abort |

---

## Generated CSV format

The output CSV is UTF-8 encoded with standard RFC 4180 quoting. It begins
with Anki import directive headers followed by one row per entry:

```
#notetype:Japanese_Mined_Vocab
#deck:Japanese::Todoku::明日は遠足
#separator:Comma
#html:true
#tags column:9
#columns:VocabID,TargetWord,Reading,Meaning,Sentence,SentenceTranslation,Notes,Audio,Tags
001_0001,始まる,はじまる,...
```

Import into Anki using **File → Import** with the option
**"Update existing notes where first field matches"** for idempotent imports.

## Adding a new Note Type

1. Create the `entries/` directory under the new Note Type folder.
2. Add an `entries-config.yaml` declaring the `notetype` and `fields`.
3. Create entry YAML files following the format above.
4. Run `python tools/yaml_to_csv.py` to generate the CSVs.
